Port 110 — POP3 (Modern)

A guide to enumeration and exploitation

<--

0) What is POP3?

POP3 (Post Office Protocol version 3) is a standard protocol used for retrieving emails from a mail server. It operates on port 110 (or 995 for POP3S with SSL/TLS). Exploiting POP3 can lead to unauthorized access to email accounts, sensitive data exposure, or credential harvesting.

1) Recon Phase 
nmap -sV -p 110,995 [targetip]
nmap --script pop3-capabilities [targetip]
nmap --script pop3-ntlm-info [targetip]

Use nmap to detect the POP3 service version and check for SSL/TLS support. Scripts like pop3-capabilities reveal server features, and pop3-ntlm-info may expose domain or authentication details if NTLM is enabled.

2) Enumeration Phase 
telnet [targetip] 110
USER [username]
PASS [password]
LIST
openssl s_client -connect [targetip]:995
hydra -L users.txt -P passwords.txt pop3://[targetip]

Connect via telnet or openssl to test for valid credentials or enumerate email lists. Use hydra for authorized brute-force testing to identify weak passwords. Be cautious with brute-forcing to avoid account lockouts.

3) Exploitation Phase 
msfconsole
use auxiliary/scanner/pop3/pop3_version
set RHOSTS [targetip]
run
use auxiliary/client/pop3/pop3_fetch
set RHOSTS [targetip]
set USERNAME [username]
set PASSWORD [password]
run

In Metasploit, scan for POP3 service details or attempt to retrieve emails with known credentials. Weak or default credentials can allow access to sensitive email content. Ensure proper authorization before testing.

POP3 Info — What to Look For
  • Unencrypted POP3 (port 110) exposing credentials in plaintext
  • Weak or default credentials for email accounts
  • Misconfigured servers allowing anonymous access
  • Outdated POP3 servers vulnerable to known exploits (e.g., buffer overflows)

Tools Required

sudo apt install nmap hydra openssl telnet
